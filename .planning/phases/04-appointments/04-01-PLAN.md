<task type="auto">
  <name>Task 3: Create Cal.com webhook endpoint</name>
  <files>app/api/webhooks/calcom/route.ts</files>
  <action>
Create app/api/webhooks/calcom/route.ts as Next.js API route:

POST handler:
1. Verify request method is POST
2. Parse webhook payload from request body
3. Handle Cal.com booking events (BOOKING_CREATED, BOOKING_CANCELLED, BOOKING_RESCHEDULED)
4. Extract booking data:
   - bookingId (uid)
   - eventId (eventTypeId)
   - startTime
   - endTime (calculate duration)
   - attendee email, phone, name
   - location, meetingUrl
   - status
5. Check for duplicate: Call getAppointmentByCalcomId(bookingId)
   - If exists and status matches, return 200 OK (idempotency)
6. Match lead: Call matchLeadToAppointment(phone, email)
   - If lead found:
     * Create appointment record with mapped fields
     * Update lead status to 'booked' via updateLeadStatusToBooked()
     * Return 201 with appointment data
   - If lead not found:
     * Log failed booking (console.log with warning)
     * Create appointment record with lead_id = null and status = 'failed'
     * Return 200 OK (don't fail the webhook)
7. Handle errors gracefully with try/catch
8. Return appropriate HTTP status codes (201, 200, 400, 500)

Cal.com webhook payload structure (for reference):
{
  "uid": "booking-id",
  "eventTypeId": "event-type-id",
  "startTime": "2024-03-04T10:00:00Z",
  "endTime": "2024-03-04T11:00:00Z",
  "attendee": {
    "email": "attendee@example.com",
    "phone": "+905551234567",
    "name": "Attendee Name"
  },
  "location": "Online",
  "metadata": {},
  "status": "ACCEPTED"
}

Security consideration: In production, verify webhook signature (Cal.com sends X-Calcom-Signature header). For now, skip signature verification (noted in TODO).

Pattern reference: Similar webhook structure as Phase 2 WhatsApp webhook (02-02).
