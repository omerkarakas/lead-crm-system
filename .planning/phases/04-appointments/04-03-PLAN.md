<task type="auto">
  <name>Task 2: Implement appointment confirmation and reminder API functions</name>
  <files>lib/api/appointments.ts, lib/api/whatsapp.ts</files>
  <action>
Extend lib/api/appointments.ts with appointment messaging functions:

1. sendAppointmentConfirmation(appointmentId: string): Promise<void>
   - Fetch appointment with lead relation
   - Format confirmation message using formatConfirmationMessage()
   - Get lead phone number from lead relation
   - Send WhatsApp message via sendWhatsAppMessage()
   - Log message to whatsapp_messages collection
   - Update appointment.confirmation_sent = true
   - Handle errors gracefully (log, don't throw)

2. send24hReminder(appointmentId: string): Promise<void>
   - Similar to above but uses format24hReminderMessage()
   - Update appointment.reminder_24h_sent = true

3. send2hReminder(appointmentId: string): Promise<void>
   - Similar to above but uses format2hReminderMessage()
   - Update appointment.reminder_2h_sent = true

4. sendCancellationNotice(appointmentId: string): Promise<void>
   - Uses formatCancellationMessage()
   - No flag update (status changed to cancelled)

5. sendRescheduledNotice(appointmentId: string): Promise<void>
   - Uses formatRescheduledMessage()
   - Reset reminder flags (reminder_24h_sent = false, reminder_2h_sent = false)

6. sendPendingReminders(): Promise<{ sent: number; errors: number }>
   - Fetch appointments needing reminders:
     * status = 'scheduled'
     * scheduled_at within next 2 hours
     * reminder_2h_sent = false
   - Send 2h reminders
   - Fetch appointments needing 24h reminders:
     * status = 'scheduled'
     * scheduled_at within next 24 hours but > 2 hours
     * reminder_24h_sent = false
   - Send 24h reminders
   - Return counts of sent and errors
   - Handle each reminder independently (one failure doesn't stop others)

7. cancelScheduledReminders(appointmentId: string): Promise<void>
   - Called when appointment cancelled or completed
   - No actual reminder cancellation needed (just won't send due to status check)
   - For future: could integrate with job queue to cancel scheduled jobs

Extend lib/api/whatsapp.ts if needed (should already have sendWhatsAppMessage and logWhatsAppMessage from Phase 2).

Pattern reference: Similar WhatsApp sending pattern as Phase 2 poll sending (02-02).
