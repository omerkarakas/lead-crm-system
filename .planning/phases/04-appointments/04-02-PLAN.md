  <files>app/api/appointments/route.ts, app/api/appointments/[id]/route.ts, lib/api/appointments.ts</files>
  <action>
Extend lib/api/appointments.ts with additional functions:

1. fetchAppointments(params: FetchAppointmentsParams): Promise<AppointmentsResponse>
   - Supports pagination (page, perPage)
   - Supports filtering by date range (startDate, endDate)
   - Supports filtering by status
   - Supports search by lead name/phone/email
   - Sort by scheduled_at DESC (default)
   - Returns paginated response with totalItems

2. fetchAppointment(id: string): Promise<Appointment>
   - Get single appointment by ID
   - Expands lead relation for display

3. updateAppointment(id: string, data: UpdateAppointmentDto): Promise<Appointment>
   - Update appointment fields
   - Handle status transitions (scheduled -> cancelled, scheduled -> rescheduled)

4. deleteAppointment(id: string): Promise<void>
   - Soft delete or hard delete (use hard delete for now)

5. sendAppointmentConfirmation(appointmentId: string): Promise<void>
   - Send WhatsApp confirmation message for appointment
   - Mark confirmation_sent = true
   - Reuse WhatsApp sending from lib/api/whatsapp.ts

Create app/api/appointments/route.ts:
- GET: List appointments with pagination and filtering
  - Call fetchAppointments()
  - Return 200 with paginated data
- POST: Create new appointment (manual)
  - Validate request body
  - Call createAppointment()
  - If lead provided, update lead status to 'booked'
  - Send WhatsApp confirmation (call sendAppointmentConfirmation())
  - Return 201 with created appointment

Create app/api/appointments/[id]/route.ts:
- GET: Get single appointment by ID
  - Call fetchAppointment()
  - Return 200 with appointment data
- PATCH: Update appointment
  - Validate request body
  - Call updateAppointment()
  - If status changed to cancelled/completed, handle reminder cleanup
  - Return 200 with updated appointment
- DELETE: Delete appointment
  - Call deleteAppointment()
  - Return 204

Pattern reference: Similar CRUD structure as leads API (01-03a).
