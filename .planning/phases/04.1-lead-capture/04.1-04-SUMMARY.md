## User Setup Required

### PocketBase Configuration for Webhook

**External services require manual configuration.** The Meta Ads webhook requires PocketBase to be configured to allow unauthenticated access (or use admin credentials):

**Option A: Configure API Rules (Recommended for Production)**
1. Open PocketBase Admin UI: http://127.0.0.1:8090/_/
2. Navigate to Settings → API Rules → leads collection
3. Configure rules to allow CREATE operation for unauthenticated requests
4. Example rule (adjust based on security requirements):
   ```
   Create: id = @request.auth.id || true
   ```

**Option B: Use Admin Authentication**
1. Create environment variable: `POCKETBASE_ADMIN_EMAIL` and `POCKETBASE_ADMIN_PASSWORD`
2. Update webhook to authenticate as admin before processing

**Option C: Service Account**
1. Create a service account in PocketBase with limited permissions
2. Use service account credentials in webhook

**Verification:**
After configuration, test with:
