## Issues Encountered

### Issue 1: Missing Company Field
**Error:** `{"success":false,"message":"Missing required fields"}`

**Cause:** Facebook Lead Ads forms don't always include `company` field

**Fix:** Added default company value:
```typescript
company: company || name.split(' ')[0] + ' (Facebook)'
```

### Issue 2: Empty UTM Params Causing Validation Errors
**Error:** `{"success":false,"message":"Processing failed","error":"Failed to create record."}`

**Cause:** Empty strings (`""`) for `utm_campaign`, `utm_content` when `adgroup_id`/`form_id` not provided

**Fix:** Only add UTM params if they have values:
```typescript
const utmParams: Record<string, string> = {
  utm_source: 'facebook',
  utm_medium: 'lead_ad',
  utm_timestamp: new Date().toISOString(),
};
if (payload.adgroup_id) utmParams.utm_campaign = payload.adgroup_id;
if (payload.form_id) utmParams.utm_content = payload.form_id;
```

### Issue 3: createdBy Relation Field Validation
**Error:** PocketBase validation error on `createdBy` field (relation to users)

**Cause:** PocketBase `createdBy` field had `nonempty` property set, causing validation failures when no user context

**User Action:** Removed `nonempty` property from `createdBy` field in PocketBase Admin UI

**Resolution:** ✅ Webhook creates leads without user context

### PocketBase Collection Context Error (RESOLVED)
During testing, discovered that PocketBase requires collection context configuration. This was resolved by:
1. Using unauthenticated PB instance (`getUnauthenticatedPB()`)
2. User removing `nonempty` property from `createdBy` field in PocketBase Admin UI

**Current status:** ✅ RESOLVED - Webhook is fully functional
