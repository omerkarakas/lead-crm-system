## Decisions Made

- **UTM fields optional:** Not all web traffic will have marketing attribution parameters
- **Duplicate matching logic:** Phone OR email matching provides flexibility for different submission scenarios
- **Duplicate update strategy:** Store old values in message field with Turkish date format "TEKRAR BASVURU [DD.MM.YYYY]"
- **Status management for duplicates:** Set status to 're-apply', reset qa_completed to false for re-qualification
- **Honeypot field naming:** Used 'fax_number' as legitimate-sounding name to attract bots
- **Honeypot response strategy:** Return success to submissions with filled honeypot to not reveal mechanism
