## Decisions Made

1. **No database migration needed** - The app_settings collection already has all required fields (service_name, setting_key, setting_value, description, is_active), so adding a new setting only requires inserting a new record, not modifying the schema.

2. **In-memory caching for booking link** - Added caching in getBookingLink() to avoid repeated database queries. Cache can be cleared with clearBookingLinkCache() after updates.

3. **Fallback to default booking link** - If the setting doesn't exist in the database or there's an error, the function returns the default URL (https://cal.mokadijital.com/moka/30min) for resilience.

## User Setup Required

**Manual configuration required:** Admin needs to create the `booking_link_url` setting in the app_settings collection. Options:

1. **Via Admin UI:**
   - Navigate to http://localhost:3000/admin/settings
   - Click "Cal.com" tab
   - Click "Yeni Ayar Ekle" (if available) or use the booking_link_url field

2. **Via API:**
   ```bash
   POST /api/settings
   {
     "service_name": "calcom",
     "setting_key": "booking_link_url",
     "setting_value": "https://cal.mokadijital.com/moka/30min",
     "description": "Randevu booking link for qualified leads",
     "is_active": true
   }
   ```

3. **Via seed script:**
   ```bash
   node scripts/seed-booking-link.mjs
   ```
   (Requires POCKETBASE_ADMIN_EMAIL and POCKETBASE_ADMIN_PASSWORD in .env.local)

**Note:** Until the setting is created, the system will use the default booking link from QA_CONFIG.defaultBookingLink.
