  <action>
    Create new webhook endpoint at app/api/webhooks/meta-ads/route.ts:

    1. Implement POST handler for Facebook Lead Ads webhook:

    Facebook Lead Ads payload structure (reference Facebook documentation):
    ```json
    {
      "leadgen_id": "123456789",
      "adgroup_id": "234567890",
      "form_id": "345678901",
      "created_time": "2026-03-05T10:00:00+0000",
      "field_data": [
        {"name": "full_name", "values": ["Ahmet Yılmaz"]},
        {"name": "phone_number", "values": ["+905551234567"]},
        {"name": "email", "values": ["ahmet@example.com"]},
        {"name": "company", "values": ["Şirket A.Ş."]},
        {"name": "website", "values": ["example.com"]},
        {"name": "message", "values": ["Interest in services"]}
      ]
    }
    ```

    2. Extract and transform field_data to internal lead model:
       - Map Facebook field names to internal fields
       - full_name → name
       - phone_number → phone
       - email → email
       - company → company
       - website → website
       - message → message (or custom field)

    3. Set source tracking:
       - source: 'web_form' (use existing enum, web form includes Meta Ads)
       - utm_source: 'facebook' or 'meta'
       - utm_medium: 'lead_ad'
       - utm_campaign: extract from adgroup_id or form_id if available

    4. Security considerations (Claude's discretion per context):
       - Option A: Verify X-Hub-Signature header (recommended for production)
       - Option B: Simple API key/secret in query param or header
       - Option C: Rate limiting based on source IP
       - Implement at minimum: Basic validation that required fields exist

    5. Error handling:
       - Log full payload for debugging
       - Return 200 OK even for malformed data (Facebook retries on 5xx)
       - Return 400 for validation errors (don't retry)
       - Return 401/403 for authentication failures

    6. Response format:
       ```json
       {
         "success": true,
         "lead_id": "abc123",
         "action": "created" | "updated"
       }
       ```

    Reference existing Cal.com webhook (app/api/webhooks/calcom/route.ts) for patterns.
