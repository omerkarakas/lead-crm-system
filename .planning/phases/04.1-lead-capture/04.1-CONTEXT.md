# Phase 4.1: Lead Capture & Pipeline Automation - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Public lead form submission, configurable booking link, and Meta Ads (Facebook Lead Ads) webhook integration for automated lead capture from multiple sources.

**Out of scope (moved to Phase 4.2):**
- Status automation based on appointment completion (now linked to proposal workflow)
</domain>

<decisions>
## Implementation Decisions

### Public Lead Form
- **Form fields:** name, phone, email, company, website, message (all standard lead fields)
- **Validation:** Same validations as existing lead modal in admin panel
- **URL:** `/lead-form` (public route, no authentication required)
- **Success behavior:** Show confirmation message, optionally redirect

### Booking Link Settings
- **Location:** Admin Settings page (new tab/section within existing settings UI)
- **Storage:** `app_settings` collection with `booking_link_url` field (single string value)
- **Validation:** Accept both full URL (https://cal.com/...) and relative path (/booking/...)
- **Hardcoded removal:** Delete `QA_CONFIG.calcomMeetingUrl` from code, use only database value

### Meta Ads Webhook (Facebook Lead Ads)
- **Endpoint:** Integrated with existing `POST /api/leads` endpoint
- **Source tracking:** Use UTM-style parameters for tracking (utm_source, utm_campaign, utm_medium, etc.)
- **Lead source value:** `LeadSource.WEB_FORM` + utm_source metadata (not new enum)
- **Duplicate handling:**
  - If phone/email matches existing lead: UPDATE existing record
  - Append old values to notes field with timestamp
  - Set lead status to `re-apply` (NEW status value)
- **Security:** Claude's discretion (balance between security and ease of use)

### Status Automation Changes
- **MOVED to Phase 4.2** - Status transitions now linked to proposal workflow:
  - Proposal sent → `CEVAP_BEKLENIYOR`
  - Lead responds → `KABUL` or `RED`
  - 7 days no response → Auto `RED`
- **NEW status added:** `re-apply` (for duplicate leads from Meta Ads)

### Claude's Discretion
- Meta Ads webhook security implementation (signature verification vs rate limiting vs simple secrets)
- Public form success message content and styling
- UTM parameter storage structure (JSON field vs individual columns)
- Form submission rate limiting strategy (if any beyond spam protection)

</decisions>
