    2. Create lib/api/sequence-executor.ts with execution engine:

       // Core execution functions
       async function startSequence(pb, enrollment_id): Initialize sequence, create message record for first step
       async function processNextStep(pb, enrollment_id): Process current step for enrollment
       async function sendStepMessage(pb, enrollment, step): Send message based on step type
       async function scheduleNextStep(pb, enrollment, delay_minutes): Calculate next step time, update enrollment

       // Delay calculation
       function calculateNextStepTime(delay_minutes, delay_type, scheduled_time?): Calculate absolute time for next step
         - If delay_type === 'relative': current_time + delay_minutes
         - If delay_type === 'absolute': use scheduled_time (already absolute)
       function getRelativeDelayMinutes(delay_value, delay_unit): Convert delay to minutes
       function validateDelaySettings(step): Check if delay configuration is valid
         - Relative: delay_minutes must be > 0
         - Absolute: scheduled_time must be in the future

       // Template variable substitution
       function substituteVariables(template, lead): Replace {name}, {company}, etc. in template
       Reference lib/email/template-variables.ts for patterns

       // Message sending
       async function sendEmailStep(pb, lead, template_id): Send email message
       async function sendWhatsAppStep(pb, lead, template_id): Send WhatsApp message

       // Status tracking
       async function createSequenceMessage(pb, enrollment_id, step): Create pending message record
       async function updateSequenceMessageStatus(pb, message_id, status, error?): Update message status
       async function getExecutionHistory(pb, enrollment_id): Get all messages for enrollment

       // Error handling (fire-and-forget)
       async function executeStepWithErrorHandling(pb, enrollment, step): Execute step, log errors but don't throw
  </action>
