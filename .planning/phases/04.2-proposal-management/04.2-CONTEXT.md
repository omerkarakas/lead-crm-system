# Phase 4.2: Proposal Management - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Sales can create proposal templates with variables, generate proposals from leads, send via WhatsApp, and track lead responses (accept/reject). Appointment completion triggers lead status → CUSTOMER or LOST based on proposal response.

**Success Criteria (from ROADMAP.md):**
1. Admin can create proposal templates with variable support ({{company}}, {{budget}}, etc.)
2. Sales can generate proposal from template and send unique link to lead via WhatsApp
3. Lead's response status is tracked (CEVAP_BEKLENIYOR / KABUL / RED)
4. Lead records show offer_document_url, offer_date, offer_response, offer_responded_at
5. When appointment completes, lead status auto-updates to CUSTOMER or LOST (based on proposal response)

</domain>

<decisions>
## Implementation Decisions

### Template Editor & Variables
- **Editor type:** Both TipTap (WYSIWYG) and Markdown supported — admin chooses
- **Variable system:** Custom-definable variable system — admin can add new variables beyond defaults
- **Formatting:** Advanced formatting (tables, images, rich text) supported
- **Variable syntax:** Single brace format — `{name}`, `{company}`, `{appointment_date}` etc.
- **Template language:** Turkish only
- **Categorization:** No categories — just name and description
- **Versioning:** No versioning — only current version kept (use soft delete for recovery)
- **Helper features:** Live preview + Ready-made templates for quick start

### Proposal Delivery Format
- **View format:** Both web page and PDF download options
- **Link protection:** Time-limited temporary link with configurable expiration (default: 3 days), configurable in admin dashboard
- **Appearance:** Print-ready custom designed page
- **PDF generation:** Browser print to PDF (native, no server-side library)
- **Branding:** Simple branding (logo + company name)
- **Mobile experience:** Button-optimized for mobile (accept/reject buttons easily clickable)

### Response Collection
- **Response method:** Buttons + comment field (Accept/Reject with optional explanation)
- **After response:** Auto-send notification to sales team via WhatsApp/SMS
- **Rejection reason:** Free text comment field (not categorized)
- **Response validity:** One-time response (cannot be changed)
- **Response history:** Track all response history (full audit trail)

### UI & Status Automation
- **Tab location:** Both Lead Detail page and Appointment Detail modal
- **Tab content:** Active + Historical proposals (show all)
- **New Proposal button:** Available in both locations
- **Status update timing:** Update immediately upon response (don't wait for appointment completion)
- **Manual override:** Role-based — only Admin can override automatic status, Sales cannot
- **Notifications:** Sales team notification when lead accepts/rejects proposal
- **Proposal list view:** Table format (not cards or timeline)

### Claude's Discretion
- Exact styling and spacing of proposal viewing page
- Color scheme and visual hierarchy of response buttons
- Notification message wording
- Error handling for expired links
- Loading states and skeleton screens

</decisions>
