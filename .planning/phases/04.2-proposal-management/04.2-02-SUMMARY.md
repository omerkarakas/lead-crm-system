## Decisions Made

- Used {variable} syntax for template variable substitution (consistent with email templates pattern from Phase 3)
- 3-day default expiration for proposal links with configurable option
- Fire-and-forget pattern for WhatsApp errors - log error but don't throw to prevent cascade failures
- Turkish language for all user-facing messages and WhatsApp notifications
- Used "secondary" badge variant for "kabul" status since no "success" variant exists in Badge component
- 30-second auto-refresh interval for proposal response updates (matches WhatsApp/Email tabs)
- Public proposal page accessible without authentication - token serves as access control
- Lead response updates both proposal record (response, response_comment, responded_at) and lead record (offer_response, offer_responded_at)
- WhatsApp notification to sales team (configurable SALES_WHATSAPP_NUMBER env var) on lead response
