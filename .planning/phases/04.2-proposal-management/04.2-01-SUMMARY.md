## Decisions Made

- **TipTap and Markdown support**: Both editor types supported to accommodate different user preferences and existing content
- **Custom variable system**: JSON field stores variable definitions (name, label, description, default_value) for extensibility beyond standard lead fields
- **Soft delete pattern**: is_deleted flag enables template recovery instead of permanent deletion
- **Turkish formatting**: Appointment dates/times formatted with tr-TR locale for consistency with rest of application
- **Variable syntax**: Both {{variable}} and {variable} syntax supported for flexibility and backward compatibility
- **Permission-based access**: Only admin role can manage proposal templates, enforced at both navigation and page level
