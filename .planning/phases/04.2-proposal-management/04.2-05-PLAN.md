<objective>
Fix sales team phone notification approach by removing broken fallback chain and making app_settings the single source of truth.

**Purpose:** The current notification system has a broken fallback chain (users.phone → app_settings → env var) that doesn't work because users.phone field doesn't exist. This plan simplifies to use ONLY app_settings for sales team phones, adds validation to prevent sending proposals without configured phones, and improves UI to make this requirement clear.

**Output:**
- Simplified notification system (single source of truth)
- Pre-send validation for proposal notifications
- Enhanced settings UI with required field indicators
- Removal of broken users.phone dependency and env var fallback

</objective>
